# Data model

All knowledge lives in `packages/core/src/data/` as JSON. Types are in
`packages/core/src/types.ts`. Contributions = edits to these files (see
[CONTRIBUTING](../CONTRIBUTING.md)).

## machines.json — `Machine[]`
`id`, `name`, `category` (`diode|co2|fiber|infrared|hybrid`), `laser_power_w[]`,
`work_area_mm` (`{width,height}` or `null` for bed-less machines like the welder),
`max_pass_through_thickness_mm`, `supported_software[]` (`XCS`/`Studio`),
`connectivity[]`, `driver` (`CH340|GD32|RNDIS|null`), `key_features[]`,
`release_era`, `product_url`, `notes`, `sources[]`.

## materials.json — `MaterialSetting[]`
`machine`, `laser_power_w`, `material`, `thickness_mm`, `operation`
(`cut|engrave|score`), `power_pct`, `speed_mm_s`, `passes`, `dpi`, `source_url`,
`confidence` (`high|medium|low|community`), `notes`.

> Every row needs a `source_url` and honest `confidence`. `high` = official published
> value; `medium` = derived/adjacent official; `low`/`community` = community-reported.
> These are starting points, not guarantees.

## troubleshooting.json — `TroubleshootingItem[]`
`id`, `symptom`, `machines[]` (empty = general), `causes[]`, `fixes[]` (ordered,
safest first), `source_url`, `tags[]`.

## guides.json — `Guide[]`
`id`, `title`, `software` (`XCS|Studio|general`), `url`, `summary`, `content` (markdown).

## embeddings.json — `EmbeddingIndex`
`{ model, dims, vectors: [{ id, vector }] }`. Regenerated by
`pnpm build:embeddings` (Workers AI). Optional — search falls back to keyword.
